# 로딩 UI 테스트 가이드

## 문제
로딩 화면이 너무 빨리 지나가서 확인하기 어렵습니다.

---

## 방법 1: 개발자 도구에서 네트워크 속도 제한 (권장)

### Chrome/Edge:
1. **F12** 키를 눌러 개발자 도구 열기
2. **Network** 탭 클릭
3. 상단의 **"No throttling"** 드롭다운 클릭
4. **"Slow 3G"** 또는 **"Fast 3G"** 선택
5. 페이지 새로고침 (F5)

### Firefox:
1. **F12** 키를 눌러 개발자 도구 열기
2. **Network** 탭 클릭
3. 상단의 **"No throttling"** 드롭다운 클릭
4. **"Slow 3G"** 선택
5. 페이지 새로고침 (F5)

이제 로딩 화면이 명확하게 보일 것입니다!

---

## 방법 2: 테스트용 지연 추가 (개발 환경)

개발 환경에서만 작동하는 테스트용 지연을 추가할 수 있습니다.

**주의**: 프로덕션에는 배포하지 마세요!

---

## 방법 3: 캐시 비활성화

1. **F12** 키를 눌러 개발자 도구 열기
2. **Network** 탭 클릭
3. **"Disable cache"** 체크박스 선택
4. 페이지 새로고침 (Ctrl + Shift + R)

---

## 방법 4: 강제로 로딩 상태 표시

브라우저 콘솔에서 다음 명령어 실행:
```javascript
// 로딩 상태 강제 표시 (테스트용)
document.querySelector('main')?.remove();
location.reload();
```

---

## 확인 사항

로딩 화면에서 다음 요소들이 보여야 합니다:
- ✅ 헤더 스켈레톤 (제목, 부제목)
- ✅ 버튼 스켈레톤
- ✅ 인증 상태 스켈레톤
- ✅ 메인 신호 카드 스켈레톤 (원형, 텍스트)
- ✅ 설명/업그레이드 프롬프트 스켈레톤

---

## 권장 방법

**방법 1 (네트워크 속도 제한)**이 가장 좋습니다:
- 실제 사용자 경험과 유사
- 추가 코드 수정 불필요
- 언제든지 원래대로 복원 가능
