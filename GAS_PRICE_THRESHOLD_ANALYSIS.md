# 유가 RISK 기준 개선 분석

## 현재 기준
- **주간 5% 이상 상승** → RISK
- **연속 3주 이상 상승** → RISK

## 문제점
1. **너무 민감함**: 주간 5% 상승은 실제로 자주 발생할 수 있음
   - 계절적 변동 (여름/겨울)
   - 공급 충격 (일시적)
   - 정상적인 시장 변동
2. **과도한 경고**: 잦은 RISK 신호로 인해 신뢰도 하락
3. **사용자 피로**: 너무 자주 경고하면 무시하게 됨

---

## 개선 방안

### 방안 1: 임계값 상향 조정 (추천)
**변경:**
- 주간 5% → **8% 이상 상승** → RISK
- 연속 3주 → **연속 4주 이상 상승** → RISK

**장점:**
- 더 의미 있는 변화만 감지
- 불필요한 경고 감소
- 신뢰도 향상

**단점:**
- 진짜 중요한 변화를 놓칠 수 있음 (하지만 연속 상승 기준으로 보완)

---

### 방안 2: AND 조건 (더 엄격)
**변경:**
- 주간 5% 이상 상승 **AND** 연속 2주 이상 상승 → RISK
- 또는 주간 10% 이상 상승 → RISK (단독)

**장점:**
- 진짜 지속적인 문제만 감지
- 일시적 변동 무시

**단점:**
- 급격한 단발성 상승을 놓칠 수 있음

---

### 방안 3: 가중치 시스템
**변경:**
- 주간 5-7% 상승: 경고 (CAUTION)
- 주간 7-10% 상승: RISK
- 주간 10% 이상 상승: 강한 RISK

**장점:**
- 더 세밀한 판단
- 단계적 경고

**단점:**
- 복잡도 증가
- 현재 시스템은 OK/RISK만 있음

---

### 방안 4: 계절 조정
**변경:**
- 계절별로 다른 임계값
- 예: 여름철(5-9월)은 7%, 겨울철은 5%

**장점:**
- 계절적 변동 고려

**단점:**
- 복잡도 증가
- 구현 난이도 높음

---

## 추천: 방안 1 (임계값 상향)

### 변경 사항:
```typescript
// 현재
if (changePercent > 5) {
  return 'risk';
}

// 개선
if (changePercent > 8) {
  return 'risk';
}

// 연속 상승도 강화
if (consecutiveIncreases >= 3) {  // 2 → 3
  return 'risk';
}
```

### 이유:
1. **8% 상승**은 실제로 의미 있는 변화
2. **연속 4주 상승**은 지속적인 추세
3. 구현이 간단하고 효과적
4. 불필요한 경고 감소

## 비교

| 기준 | 현재 | 개선안 1 | 개선안 2 |
|------|------|----------|----------|
| 주간 상승 | 5% | 8% | 10% (단독) 또는 5%+연속2주 |
| 연속 상승 | 3주 | 4주 | 2주 (AND 조건) |
| 민감도 | 높음 | 중간 | 낮음 |
| 신뢰도 | 낮음 | 높음 | 매우 높음 |
