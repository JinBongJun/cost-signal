# 완성도 향상 우선순위 로드맵

## 현재 상태 분석

### ✅ 완료된 기능
- 기본 신호 표시 (OK/CAUTION/RISK)
- 데이터베이스 (Supabase)
- API 엔드포인트 (signal, history, cron)
- 로그인/회원가입 페이지 (UI)
- 푸시 알림 코드
- 히스토리 기능 코드

### ❌ 미완성 기능
- 인증 시스템 작동 (NEXTAUTH 환경 변수 없음)
- 주간 데이터 자동 업데이트
- 에러 처리 개선
- 로딩 상태 개선
- 사용자 경험 개선

## 우선순위별 작업 목록

### 🔴 1순위: 핵심 기능 완성 (즉시 필요)

#### 1.1 인증 시스템 활성화
**왜 중요?**
- 로그인/회원가입이 작동하지 않음
- 유료 기능 사용 불가
- 사용자 데이터 저장 불가

**작업 내용:**
- [ ] Vercel에 `NEXTAUTH_SECRET` 추가
- [ ] Vercel에 `NEXTAUTH_URL` 추가
- [ ] 로그인/회원가입 플로우 테스트
- [ ] 세션 관리 확인

**예상 시간:** 10분

#### 1.2 주간 데이터 자동 업데이트 설정
**왜 중요?**
- 현재 수동으로만 업데이트 가능
- 매주 자동으로 새 데이터 가져와야 함
- 사용자 경험 핵심

**작업 내용:**
- [ ] Vercel Cron Jobs 설정 확인
- [ ] `vercel.json` cron 설정 확인/수정
- [ ] 테스트 실행
- [ ] 에러 알림 설정 (선택)

**예상 시간:** 15-20분

---

### 🟡 2순위: 사용자 경험 개선 (중요)

#### 2.1 에러 처리 개선
**왜 중요?**
- API 실패 시 사용자에게 명확한 메시지 필요
- 네트워크 오류 처리
- 데이터 없을 때 안내

**작업 내용:**
- [ ] API 에러 메시지 개선
- [ ] 재시도 로직 추가
- [ ] 사용자 친화적 에러 메시지
- [ ] 폴백 UI 추가

**예상 시간:** 30-40분

#### 2.2 로딩 상태 개선
**왜 중요?**
- 현재 로딩 상태가 명확하지 않을 수 있음
- 사용자에게 진행 상황 표시 필요

**작업 내용:**
- [ ] 스켈레톤 로딩 추가
- [ ] 진행률 표시 (선택)
- [ ] 로딩 애니메이션 개선

**예상 시간:** 20-30분

#### 2.3 히스토리 기능 UI 개선
**왜 중요?**
- 히스토리 API는 있지만 UI가 부족할 수 있음
- 과거 신호를 시각적으로 보여주기

**작업 내용:**
- [ ] 히스토리 섹션 UI 개선
- [ ] 타임라인 표시
- [ ] 필터링 기능 (선택)

**예상 시간:** 1-2시간
